# m-wap-pages

## 项目简介

m-wap-pages 是一个用于归档和发布独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态页面资源的统一存储与版本管理空间。

仓库中的每个页面均为独立的 HTML 文件，可单独访问或嵌入其他项目使用。适合存放活动页、工具页、说明页等轻量级前端页面。

## 目录结构

```
m-wap-pages/
├── pages/          # 存放所有 HTML 页面文件
│   ├── page-a.html
│   ├── page-b.html
│   └── ...
├── assets/         # 共用的静态资源（CSS、JS、图片等）
├── index.html      # 仓库首页（可选，用于页面列表导航）
└── README.md       # 本文件
```

- `pages/`：页面主体目录，每个文件对应一个独立页面。
- `assets/`：存放多个页面共用的样式、脚本或图片资源，便于统一维护。
- `index.html`：可选的入口页面，用于展示所有已归档页面的链接列表。

## 页面归档说明

- 所有页面均为纯 HTML 实现，不依赖后端服务。
- 页面之间相互独立，命名以描述性英文或拼音为主，避免歧义。
- 如需引用外部资源（如字体、图标），建议使用 CDN 或放入 `assets/` 目录。
- 页面内容应遵循通用的 Web 标准，确保在主流浏览器中正常显示。

## 维护说明

- 新增页面：直接在 `pages/` 目录下创建 `.html` 文件，并确保路径引用正确。
- 更新页面：修改对应文件后提交即可，建议附带简要的 commit 说明。
- 删除页面：确认不再需要后，从 `pages/` 中移除文件，并检查是否存在相关引用。
- 共用资源：若新增资源，请放入 `assets/` 下对应子目录，避免散落在页面文件夹中。

## 使用方式

可通过 GitHub Pages 或其他静态托管服务将本仓库部署为站点，此时 `pages/` 下的每个 HTML 文件均可通过对应 URL 直接访问。

本地预览时，直接在浏览器中打开 `pages/` 下的文件即可，部分功能（如跨域请求）可能受限，建议使用本地服务器（如 `python -m http.server`）进行测试。
